# 에듀테크 활용현황 정보 서비스 — 개발 핸드오프 문서

작성: 2026-07-20, 서울에듀테크소프트랩 Cowork 세션에서 이관.
이 문서는 Cowork(리서치 세션)에서 진행한 조사·논의를 Claude Code(개발 세션)로 넘기기 위한 것이다. 이 문서와 `db_export.csv`를 작업 폴더에 두고 개발을 시작할 것.

## 1. 서비스 목표 (합의된 방향)

일반인이 특정 URL에 접속해 "어떤 학교가 어떤 에듀테크를 쓰는지"와 "우리나라 전반의 에듀테크 활용 현황"을 검색·열람하는 공공 정보 서비스. 벤치마크: 국가법령정보 MCP(법령 데이터를 AI가 쓸 수 있게 구조화한 사례)의 에듀테크 판.

핵심 아키텍처 결정(논의 완료):
- 중심에 **데이터베이스 + 조회 API 서버** 하나를 두고, 출구(문)를 두 개 낸다.
- **문 1 — 웹사이트(주력, 일반인용)**: 검색창(학교명/제품명) → 결과 목록·학교 상세·제품별 채택학교, 지역·계열별 통계(지도·차트). 설치·로그인 불필요.
- **문 2 — MCP 서버(부가, 전문가·AI용)**: 같은 API를 MCP 도구로 감싼 것. 교육청·연구자·기업이 자기 AI에 연결. 웹사이트와 병렬이며 상호 의존 없음.
- 웹사이트 안 질문형 검색(챗봇)은 선택 사항. 붙일 경우 서버 내부에서 MCP/API를 부품으로 재사용 — 사용자에게 MCP는 보이지 않음.
- 우선순위: ① 웹 프로토타입(정적 대시보드) → ② 수집 자동화 정기화 → ③ 챗봇/MCP.

## 2. 데이터 자산 (이 서비스의 원료)

- `db_export.csv` — 활용사례 DB. 2026-07-20 기준 543건(분석 533건), 나라장터 용역 창 10개 수집이 완료되면 약 600~640건으로 최종화 예정(같은 프로젝트의 `claude/활용사례DB_데이터_2026-07-14.csv`가 항상 최신본).
- 컬럼(11개): 학교명, 유형(과학고/영재학교/특성화고/마이스터고), 지역, 제품명, 카테고리, 시기, 내용(계약명·금액 포함), 출처유형, URL, 신뢰도(상/중/하), 비고
- 커버리지: 과학고 계열 28개교 전수 + 특성화고·마이스터고 239개교. 제품(군) 과학고 57종·특성화고 146종(제품명 특정 기준).
- 조사 기간: 2023.1~2026.7. 출처: S2B 학교장터, 나라장터 계약정보 OpenAPI, 부산/인천/서울 교육청 계약공개, KERIS 사례집, 학교 공식자료, 언론, 학술DB.

주요 집계(현재): ChatGPT 특성화고 39개교·과학고 13개교, Adobe 특성화고 44개교, AIDT 유료구독 26개교, AI면접시스템 19개교, AI로봇·자율주행 장비 19개교, 카피킬러 과학고 11개교.

## 3. 데이터 수집 파이프라인 (자동화 대상)

현재 Cowork 세션에서 반자동으로 운영 중. 서비스화 시 서버 배치로 이식할 것.

1) **나라장터 계약정보서비스 OpenAPI** (data.go.kr/data/15129427)
   - endpoint: `https://apis.data.go.kr/1230000/ao/CntrctInfoService/getCntrctInfoList{Thng|Servc}`
   - params: serviceKey(사용자 발급 키 보유), inqryDiv=1, inqryBgnDt/EndDt(YYYYMMDDHHMM, 3개월 창), numOfRows=999, pageNo, type=json
   - 필터: 수요기관에 '고등학교' 포함 && 계약명 키워드 정규식(AI|에듀테크|구독|소프트웨어|라이선스|코스웨어|플랫폼|GPT|Adobe 등) → 직업계고·과학고 교명 필터
   - **제약: 개발계정 일일 트래픽 쿼터가 오퍼레이션별(물품/용역 각각)·일일제(KST 자정 리셋)**. 운영계정 전환 신청으로 상향 가능 — 서비스화하려면 필수.
2) **S2B 학교장터** — 폼 기반 조회(tcmo001.do), 3개월 창, 요청 간 20초 엄수(위반 시 CAPTCHA). 기관명 접미사 스윕(공업고·상업고·정보고·전자고 등)으로 교명 커버.
3) **교육청 계약공개 시스템** — 부산/인천/서울 각기 방식 상이(세션 조사노트 참조). 타 시도 미개척.
4) 보조: KERIS 디렉토리북(연도판 PDF OCR), 학교 홈페이지·언론.

## 4. 데이터 품질 원칙 (서비스 신뢰도의 핵심)

- 병합·수집 시 계약번호·시기·소재지 재검증 (실제 오귀속 4건·기간외 3건·지역오류 1건을 잡아낸 이력 있음. 서비스는 학교 실명이 공개되므로 오귀속은 치명적 — 검증 단계 자동화 필요)
- S2B↔나라장터 금액 교차검증 4건 확보(데이터 정합성 근거)
- 명시할 한계: 본 데이터는 **하한 추정치** — 제품명 미확인 계약(별도 그룹 관리), OpenAI 해외 직접결제 등 조달 외 경로, 사비 구매는 미포착. 서비스 화면에 한계 고지 필수.
- 표기 규칙(확정): 학교명 — 목록·표는 "-고" 단축형(영재학교 등 예외), 문장은 "-고등학교". 제품명 — 외국산·영문 브랜드는 영어(ChatGPT, Adobe, Padlet), 한글 브랜드 국산은 한글(영문) 병기(카피킬러, 리로스쿨).

## 5. MVP 제안 (Claude Code에서 시작할 것)

**1단계 — 정적 프로토타입** (현 CSV만으로 가능, 배포는 정적 호스팅):
- CSV → SQLite(또는 JSON 인덱스) 변환 스크립트
- 단일 페이지 앱: 검색창(학교/제품 자동완성), 학교 상세(사용 제품 타임라인), 제품 상세(채택 학교 목록·지도), 통계 대시보드(계열별·지역별·연도별)
- 한계 고지 문구 + 출처(조달 기록) 링크

**2단계 — API 서버 + 정기 수집**: FastAPI(또는 Next.js API) + 수집 배치(위 파이프라인 이식, 쿼터 관리 포함) + 신규 사례 검증 큐
**3단계 — MCP 서버**: 조회 API 위에 도구 3~4개(searchSchool, searchProduct, getStats, getTrends) 노출. 선택: 사이트 내 챗봇.
